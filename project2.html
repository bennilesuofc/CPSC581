Touch/Gesture-Based Unlock

Press-and-Hold to Unlock

demo video

Link to Demo Link to Source Code

I came up with this idea by considering the types of input possible on a touchscreen. It’s a two-dimensional surface, so I thought it might be interesting to try making an idea that uses the dimension of time as well. I chose the Press-and-Hold idea because it seemed like it would be practical as an actual unlocking mechanism, while still not being too difficult to implement.

Sensor-Based Unlock

Connect a USB Device to Unlock

demo video

Link to Demo Link to Source Code

I thought of using a physical device to unlock a digital key, and that seemed like an interesting concept. I had initially started by looking at what sensors are included on phones and can be accessed in a web development environment, but a lot of them are only partially implemented or not at all. In fact, detection of and communication with USB devices is not supported by iOS devices, and since I don’t have access to an Android device, I had to develop and test using the Chrome browser on MacOS. I looked into using XCode to create an iOS app with the required functionality, but unfortunately that development environment did not run well on my computer, so I decided to stick to HTML and JavaScript. Some other issues I ran into were that some USB devices don’t have serial numbers and are forgotten by the browser on disconnect, while some USB devices aren’t recognized by the browser at all.
